# 软件简介 
软件主要功能是通过读取模型文件 brep stp 或者手动创建几何后可以通过 opengl 渲染显示，几何使用 OCC 进行管理，还要支持用户通过鼠标编辑几何，例如 trim 等，支持选择几何点、边、面以及 part，并且后续还会对几何面进行网格剖分，最终还要引入 AI 来实现网格质量诊断与协助修复差网格信息。

# 软件架构
- include 头文件包括对外接口，不同 src 下的目录为不同模块，只能通过 include 暴露的接口进行调用

# 计划任务

1. 了解当前实现，按如下要求完成开发: 实现一个基础 qml 控件，select 控件，可以通过鼠标点击的方式选择 opengl 视口中的几何实体，支持点、边、面、体、part 以及后续的 mesh 相关多种选择模式（当前可以不实现 mesh， 但是设计要考虑后续扩展）:
    - 前端设计：
      - 当激活 select 控件后，进入拾取模式，鼠标在 opengl 渲染窗口移动时，自动高亮当前鼠标悬停的实体。当鼠标点击后变为拾取的颜色。
      - 进入拾取模式后可以多次拾取，在 select 控件上显示出当前拾取的实体信息 type:uid 列表。
      - 支持框选功能，鼠标左键按下拖动形成一个矩形框，松开后框选区域内的实体均为选中状态。
      - 支持取消选中功能，鼠标右键点击或者右键框选可以取消选中状态。
    - 后端设计：
      - 渲染层需要将几何的 entity id 信息编码到 opengl 的颜色 buffer 中，方便拾取时通过读取像素颜色获取 entity id。
      - 后端 geometry 模块实现一个 查询 action，使得前端根据 entity id 可以获取对应 entity 的所有信息。
      - 后端 geometry 实现一个标记高亮的 action，使得前端可以通过 entity id 列表标记高亮对应实体，并更新渲染信息使其高亮，分为 preview 与 select 两种状态。
      - 所有协议支持前端框选时传入一个 entity id 列表进行批量操作。
      - 注意所有的实现不要破坏现有的架构设计，尽量使用 json 协议进行模块间通信，不要随意新增 qt service 组件与 qml 通信。
2. 在 Geometry Tab 中新增一个 Query 界面，icon 是一个放大镜，应用拾取组件，可以拾取信息并显示在界面上，注意控件与界面的美化风格要统一。
